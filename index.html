A transaction is a unit of program execution that accesses and or possibly updates various data items. Acid properties of a transaction are :
Atomicity --- either transaction executes in entirety or none executes at all
Consistency --- state of database remains same both before & after transaction despite system changes
Isolation ---- despite concurrent execution of transactions , each transaction proceeds independently of others despite system changes
Durability ----- any modifications made to database as a result of transaction persist even if there are system changes
